PABW Shop — Navigation Redesign

Unified navigation architecture for an AI-powered marketplace. 3 directions exploring different trade-offs between discoverability, density, and brand expression.

Problems

  • Home page has its own header — other pages use a different Navbar
  • Bottom bar only exists on home page; mobile users on PDP/cart get lost
  • 6 tabs on mobile = cramped touch targets below 44px
  • Bottom bar shows on desktop too — wastes vertical space
  • AI assistant entry is a tiny icon — poor discoverability
  • Notification bell in Navbar is inert (no dropdown for regular users)
  • Search only available on home page header

Proposed Solution

  • Unified Navbar component across ALL buyer pages (home + PDP + cart + profile)
  • Responsive: mobile = compact top bar + 4-tab bottom bar; desktop = full top nav
  • 4 primary tabs + floating AI FAB (replaces 6th tab, elevates assistant)
  • Desktop: no bottom bar; top nav carries all navigation
  • AI entry point becomes a prominent FAB or dedicated tab
  • Notification bell → contextual badge + simple slide-down panel
  • Search moves into the unified Navbar (collapsible on mobile)

3 Design Directions

Each direction reimagines the navbar + bottom bar as a unified system. Click "Open Prototype" for an interactive demo.

Motion & Interaction Notes

Navbar → scroll
On scroll down: top nav shrinks 8px, shadow deepens. On scroll up: springs back.

Bottom bar → hide on scroll
Scroll down = bar slides down (hidden). Scroll up = springs back. Prevents content occlusion during reading.
AI FAB → pulse on idle
After 8s of inactivity, the FAB does a subtle pulse. On first visit, a shimmer draws attention for 3s.

Tab switch → no flash
Active tab uses scale(1) + color fill. Inactive → scale(0.92) + muted. No full-page reload feel.
